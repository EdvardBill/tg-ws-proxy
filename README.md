# TG WS Proxy для Padavan

WebSocket MTProto Proxy сервер для роутеров с прошивкой Padavan. Позволяет развернуть собственный прокси для Telegram на домашнем роутере.

## Возможности

- **MTProto Proxy** через WebSocket transport для обхода блокировок Telegram
- **Веб-интерфейс управления** с информацией о подключении и копированием данных
- **Автозапуск** - прокси автоматически запускается при старте роутера
- **Простая установка** - один скрипт с интерактивным меню
  
## Установка

### Вариант 1: Установка с ПК через SSH

1. Скачайте `twpm.sh` на компьютер
2. Подключитесь к роутеру по SSH:
   ```bash
   ssh admin@192.168.1.1
   ```
3. Скопируйте скрипт на роутер:
   ```bash
   scp twpm.sh admin@192.168.1.1:/tmp/
   ```
4. Запустите скрипт:
   ```bash
   chmod +x /tmp/twpm.sh
   /tmp/twpm.sh
   ```
5. В меню выберите **1) Установить**

### Вариант 2: Установка напрямую на роутере

1. Скачайте `twpm.sh` напрямую на роутер:
   ```bash
   curl -sL https://raw.githubusercontent.com/EdvardBill/tg-ws-proxy/main/twpm.sh -o /tmp/twpm.sh
   chmod +x /tmp/twpm.sh
   ```
2. Запустите:
   ```bash
   /tmp/twpm.sh
   ```

## Использование

### Веб-интерфейс

После установки откройте в браузере:
```
http://192.168.1.1:8081
```

Веб-интерфейс позволяет:
- Просматривать статус прокси
- Запускать/останавливать/перезапускать сервис
- Копировать данные для подключения (хост, порт, ключ)
- Получить ссылку для быстрого добавления прокси

### Подключение в Telegram

1. Нажмите **Запустить** в веб-интерфейсе
2. В Telegram: **Настройки → Данные и память → Прокси**
3. Выберите тип: **MTProto**
4. Введите данные:
   - **Хост**: IP адрес роутера (например, 192.168.1.1)
   - **Порт**: 1443
   - **Ключ**: секретный ключ (начинается с `dd`)
5. Или нажмите на ссылку `tg://proxy?...` в веб-интерфейсе

### Управление из терминала

Подключитесь к роутеру по SSH и используйте init-скрипт:

```bash
# Запуск прокси
sh /opt/etc/init.d/S99tgwsproxy start

# Остановка прокси
sh /opt/etc/init.d/S99tgwsproxy stop

# Перезапуск прокси
sh /opt/etc/init.d/S99tgwsproxy restart
```

### Изменение секретного ключа

```bash
# Сгенерировать новый ключ
openssl rand -hex 16 > /opt/home/admin/proxy_secret.txt

# Перезапустить прокси
sh /opt/etc/init.d/S99tgwsproxy restart
```
## Диагностика

### Проверка статуса

```bash
# Проверить, запущен ли процесс
pgrep -f "proxy.tg_ws_proxy"
```

### Просмотр логов

```bash
# Логи прокси
cat /var/log/tg-ws-proxy.log

# Логи веб-интерфейса
cat /tmp/tg-ws-web.log
```
## Принцип работы

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   Telegram  │────▶│  MTProto     │────▶│  Telegram   │
│   Client    │     │  Proxy       │     │   Servers   │
└─────────────┘     │  (port 1443) │     └─────────────┘
                    └──────────────┘
                          ▲
                          │
┌─────────────┐     ┌──────────────┐
│   Browser   │────▶│  Web UI      │
│   (port 8081)     │              │
└─────────────┘     └──────────────┘
```

1. **MTProto Proxy**  - принимает подключения от клиентов Telegram через WebSocket и перенаправляет трафик на серверы Telegram
2. **Веб-интерфейс** - простой HTTP сервер для управления прокси и отображения данных подключения
3. **Init-скрипт**  - обеспечивает автоматический запуск сервисов при старте роутера

## Безопасность

- Секретный ключ генерируется автоматически при установке
- Рекомендуется использовать `dd` перед секретом (16-byte hex format)
- Веб-интерфейс доступен только в локальной сети
- При необходимости ограничьте доступ к порту 8081 в файрволе
